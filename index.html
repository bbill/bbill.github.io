2 Solution

  1. Turn off Xserver used 
    sudo init 3
  2. Remove xserver-xorg-core 
    sudo apt-get remove  xserver-xorg-core -y
  3. Install the driver or so 
    sudo apt-get install  spice-vdagent gdm xserver-xorg-video-qxl-hwe-16.04
  4. Create the xorg.conf we needed 
    sudo X -configure
sudo cp ./xorg.conf.new /etc/X11/xorg.conf
  5. Get X Back 
    sudo apt-get install   xserver-xorg-core xserver-input-all -y
  6. Make sure spice-vdagent start automatically 
    sudo systemctl enable spice-vdagentd
sudo systemctl start spice-vdagentd
  7. Time to cross your finger 
    sudo init 5

If everything goes smoothly, your screen resolution will change automatically.

    在一台机器上用dnsmasq 做dhcp server，报错 

    failed to bind dhcp server socket: Address already in use.

    查看后发现系统内已经由libvirtd 启动了一个dnsmasq，对virtbr0提供服务。这里提供了解决方案。 关键是取消 bind-interfaces这一行的注释。并且指定需要监听的interface或者IP。 

    bind-interfaces
interface=eth0
listen-address=192.168.0.1
